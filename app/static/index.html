How AAIS Works

AAIS is a nationwide emergency workflow engine designed to eliminate transit delay, ensure clinical-bed fit, and notify emergency rooms before ambulances arrive.

This simulator demonstrates a live incident moving through our 7 architectural stages, highlighting how AI-driven clinical reasoning integrates with deterministic routing rules.

Core: FastAPI, Python 3.11
AI Triage: Configurable LLM provider (LM Studio or NVIDIA NIM)
Routing: Dynamic Multi-Factor Scoring

Routing Engine Formula & Weights

AAIS maps medical indicators to hospital capabilities, filtering out hospitals lacking absolute clinical requirements (e.g. Stroke clinical pathway requires CT scanner). Reaching surviving hospitals are graded via:

Weight Metric Target Value Impact
Clinical Capability (40%) Matches triage pathways + staff Ensures specialized center assignment
Capacity Resources (25%) Available ER beds + ventilators Guarantees resources are free upon arrival
Ambulance Travel ETA (20%) GIS calculated drive ETA Minimizes travel delays
ER Load Balance (10%) Busy ratio of the Emergency Room Prevents overcrowding and ambulance queueing
Record Continuity (5%) EHR preferred hospital history Leverages preexisting clinician relationships

National Scale Production Architecture

Evolving from the AAIS Pilot Simulator to a highly available, nationwide emergency infrastructure.

🗄️

1. Data Persistence Layer

Pilot: In-Memory / Python Session Store → Production: PostgreSQL + PostGIS (GIS Spatial) + Redis Cache

Moving from ephemeral local session dicts to structured database schemas with indexing on latitude/longitude spatial coordinates, optimized for complex municipal geometries.

🚀

2. Event Streaming & Telemetry

Pilot: Periodic HTTP API polling → Production: Apache Kafka Cluster / WebSockets

Supporting real-time GPS telemetry from thousands of mobile NAS units and hospital bed sensors. Event streams ensure instantaneous UI synchronization without taxing server loads.

🔒

3. Enterprise Security & DPA Compliance

Pilot: Public routes, mock IDs → Production: OpenID Connect, AES-256 Patient Cryptography

Full integration with Keycloak/Auth0 for Role-Based Access Controls. Encryption-at-rest for sensitive clinical patient records matching the strict rules of the Ghana Data Protection Act.

🔌

4. Integrated National APIs

Pilot: Mocked JSON Seeding → Production: GH Card Registry API + NHIS Eligibility Gateway

Direct connectivity with the National Identification Authority (NIA) to pull verified demographic snapshots via fingerprints or card numbers, linking patients automatically with national insurance eligibility profiles.

🤖

5. Clinical LLM Guardianship

Pilot: LM Studio or NVIDIA NIM provider → Production: Redundant LLM Ensemble + Human Verification

Triage prompts route to parallel specialized clinical models with fallback strategies. High-acuity decisions require physical dispatcher confirmation on confidence drop, avoiding model hallucination risks.

📟

6. ER ERX pre-alert delivery

Pilot: Web-socket telemetry rendering → Production: WhatsApp ER Dispatch Bot, SMS & EHR FHIR Push

Ensuring ER clinicians get alerts immediately by broadcasting to critical clinical WhatsApp groups, Twilio SMS fallback channels, and writing directly to their internal electronic hospital health record systems.
